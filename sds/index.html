2. Architectural Design

Model–view–controller (MVC) is an architectural pattern commonly used for developing user interfaces that divides an application into three interconnected parts. This is done to separate internal representations of information from the ways information is presented to and accepted from the user.

We are using django to develope our website.

Django follows the MVC pattern closely enough to be called an MVC framework. Django has been referred to as an MTV framework because the controller is handled by the framework itself and most of the excitement happens in models, templates and views. In Django’s interpretation of MVC, the view describes the data that gets presented to the user; it’s not necessarily just how the data looks, but which data is presented.

  • • UI: It is placeholder for boundary classes.
  • • Controller: Controller classes are responsible for controlling the interaction between user and system.
  • • Product: It provides the information about all products available. Also allows searching them, manages product categories.
  • • Shopping cart manage: It allows the users to manipulate the contents of shopping cart and products and place products into them.
  • • User management: It tracks information about all types of users and their accounts. It will encapsulate the domain classes- Users, Customers, Employees, Admins, along with classes that will be used to manage various types of user accounts.
  • • Order manage: It performs order processing when user select product(s), add to cart and buys that/those product(s) which is/are in cart. This will also contain payment method and shipping system to gather the information on the state of delivery.
